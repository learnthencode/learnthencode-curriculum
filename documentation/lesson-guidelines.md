# LearnThenCode Lesson Guidelines

## Purpose

This document defines the standard structure and quality expectations for LearnThenCode lessons.

Every lesson should provide learners with clarity, context, and practical experience.

---

# Lesson Structure

Each lesson should contain:

## Title

A clear and descriptive lesson title.

---

## Overview

A short explanation of what the learner will study.

---

## Learning Objectives

By the end of the lesson, learners should be able to:

- Explain concepts.
- Apply knowledge.
- Complete practical tasks.

---

## Why This Lesson Matters

Explain the real-world importance of the topic.

Learners should understand why they are learning something before implementing it.

---

## Lesson Content

Content should be divided into smaller sub-lessons when appropriate.

Structure:

```
Course
 └── Lesson
      └── Sub-lessons
```

---

## Examples

Examples should demonstrate concepts clearly before learners attempt independent work.

---

## Coding Labs

Labs should reinforce understanding through practical implementation.

Labs may include:

- Completing incomplete code.
- Fixing bugs.
- Making tests pass.
- Building small features.

---

## Summary

Every lesson should conclude with the main ideas learners should remember.

---

## What's Next?

Lessons should connect learners to the next stage of their learning journey.

---

# Writing Principles

Lessons should:

- Explain concepts clearly.
- Avoid unnecessary complexity.
- Use practical examples.
- Encourage curiosity.
- Connect theory to real-world applications.

---

# Code Examples

Code examples should:

- Be readable.
- Follow professional conventions.
- Include explanations where necessary.
- Avoid unnecessary complexity.
