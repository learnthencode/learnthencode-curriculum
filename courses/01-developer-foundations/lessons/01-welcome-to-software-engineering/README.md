# Welcome to Software Engineering

## Overview

Welcome to LearnThenCode and your journey into software engineering.

Before writing code, it is important to understand the world you are entering.

Software engineers do not simply write instructions for computers. They solve problems, design solutions, collaborate with others, and build systems that impact how people live, work, and communicate.

This lesson introduces the foundations of software engineering and helps you understand the difference between writing code and becoming a software engineer.

The goal is simple:

> **Understand the field first, then begin building.**

## Why This Matters

Many beginners start learning software development by immediately writing code.

While writing code is important, professional software engineering requires much more.

A software engineer must understand:

- The problems they are solving.
- The people who use the software.
- How different parts of a system work together.
- How to create software that is reliable and maintainable.

Learning these foundations helps you become more than someone who can write code.

# Sub-lessons

## What Is Software?

Software is a collection of instructions, programs, and data that tell a computer how to perform tasks.

Unlike hardware, which refers to the physical components of a computer, software exists as instructions that can be executed by a machine.

Examples of software include:

- Web browsers.
- Mobile applications.
- Operating systems.
- Video games.
- Banking systems.
- Learning platforms.
- Business applications.

Every time you use a smartphone application, visit a website, or interact with a digital service, you are using software.

### Software Categories

Software can be grouped into different categories.

#### System Software

System software manages computer hardware and provides a foundation for other applications.

Examples:

- Windows.
- macOS.
- Linux.
- Device drivers.

#### Application Software

Application software helps users perform specific tasks.

Examples:

- Web browsers.
- Microsoft Word.
- Mobile applications.
- Video editing software.

#### Development Software

Development software helps developers create other software.

Examples:

- Code editors such as Visual Studio Code.
- Programming language tools such as Node.js.
- Version control systems such as Git.
- Debugging tools such as Chrome DevTools.

---

## What Is Software Engineering?

Software engineering is the discipline of designing, developing, testing, deploying, and maintaining software using engineering principles.

A software engineer does not only focus on making software work. They also consider:

- Reliability.
- Security.
- Performance.
- Scalability.
- Maintainability.
- User needs.

Software engineering combines:

- Programming knowledge.
- Problem-solving skills.
- Design principles.
- Collaboration.
- Continuous improvement.

A software engineer focuses not only on creating software, but also on creating software that can be trusted, improved, and maintained.

## Coding vs Programming

The terms coding and programming are often used interchangeably, but they represent different levels of activity.

### What Is Coding?

Coding is the process of writing instructions in a programming language that a computer can understand.

Example:

```javascript
console.log("Hello, LearnThenCode!");
```

This code instructs a computer to display a message.

Coding focuses on:

- Writing syntax correctly.
- Using programming languages.
- Creating instructions.

Coding is one part of programming.

### What Is Programming?

Programming is the broader process of creating solutions using code.

Programming involves:

- Understanding a problem.
- Planning a solution.
- Designing logic.
- Writing code.
- Testing results.
- Debugging errors.
- Improving the solution.

A programmer does not only ask:

> "How do I write this code?"

They also ask:

> "What problem am I solving, and what is the best way to solve it?"

### The Relationship Between Coding and Programming

A simple way to remember:

```
Coding
    ↓
Writing instructions

Programming
    ↓
Using code to solve problems
```

All programming involves coding, but not all coding represents complete programming.

---

## Software Development vs Software Engineering

These terms are closely related but represent different perspectives.

### What Is Software Development?

Software development is the process of creating software products.

It includes activities such as:

- Planning.
- Designing.
- Coding.
- Testing.
- Deploying.
- Maintaining software.

A software developer contributes to building software solutions.

### What Is Software Engineering?

Software engineering applies engineering principles to software development.

Software engineers think about:

- How systems should be designed.
- How software can handle growth.
- How teams can collaborate effectively.
- How software can remain reliable over time.

### Understanding the Difference

| Software Development | Software Engineering |
| --- | --- |
| Focuses on creating software | Focuses on creating software using engineering principles |
| Often focuses on implementation | Considers the complete software lifecycle |
| May focus on individual features | Considers scalability, reliability, and maintainability |

Both are important.

Software engineering builds upon software development by introducing engineering practices that help teams create better software.

## The Software Engineer Mindset

Becoming a software engineer requires more than learning programming languages.

A software engineer develops habits and ways of thinking that support effective problem-solving.

### Curiosity

Great engineers ask questions:

- Why does this work?
- How does this system operate?
- Is there a better approach?

### Problem Solving

Software engineering is about solving problems.

Instead of immediately writing code, engineers:

1. Understand the problem.
2. Break it into smaller parts.
3. Design a solution.
4. Implement and test.

### Continuous Learning

Technology changes constantly.

Successful engineers learn how to:

- Read documentation.
- Research solutions.
- Adapt to new tools.
- Improve their skills.

### Collaboration

Software is rarely built alone.

Engineers collaborate through:

- Discussions.
- Code reviews.
- Documentation.
- Team workflows.
