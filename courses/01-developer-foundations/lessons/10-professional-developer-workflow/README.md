# Professional Developer Workflow

## Overview

Throughout this course, you have learned the foundations that support professional software development.

You started by understanding what software engineering is, how computers execute software, and how developers transform problems into solutions.

You then learned how to prepare your development environment, use the command line, organize projects, track changes with Git, and collaborate using GitHub.

Professional developers do not use these skills separately.

They combine them into a structured workflow that helps them create, document, share, and maintain software projects.

In this lesson, you will bring together everything you have learned and explore how professional developers approach software projects from beginning to end.

The goal is simple:

> **Understand the workflow professional developers follow when building and maintaining software.**

# Why This Matters

Professional software development is not only about writing code.

A developer's effectiveness depends on how well they organize their work, communicate ideas, and collaborate with others.

A professional developer must know how to:

- Structure projects clearly.
- Document important information.
- Track changes safely.
- Share work with others.
- Maintain software after release.

The tools you learned throughout this course become powerful when combined into a complete workflow.

# Sub-lessons

# Starting a Software Project

Every software project begins with a problem, idea, or requirement.

Before writing code, developers first understand:

- The problem being solved.
- The users who need the solution.
- The required features.
- The expected outcome.

A simplified process:

```text
Problem
    ↓
Understanding Requirements
    ↓
Planning
    ↓
Development
    ↓
Testing
    ↓
Release
```

Starting with understanding prevents developers from building solutions that do not solve real problems.

---

# Creating Project Repositories

A repository is the home of a software project.

It contains the files, documentation, and history required to build and maintain software.

A professional repository usually contains:

```text
project-name
├── src
├── docs
├── tests
├── README.md
└── .gitignore
```

Each part has a purpose.

---

## Source Code

The `src` folder contains the main application code.

Example:
```text
src/
```
This is where developers create and organize software features.

---

## Documentation

The `docs` folder contains information that helps people understand the project.

Examples:

- Setup instructions.
- Project decisions.
- Usage guides.
- Technical explanations.

Good documentation helps both current and future developers.

---

## Tests

The `tests` folder contains automated or manual tests.

Testing helps developers confirm that software behaves correctly.

---

## README.md

The README file introduces the project.

A good README explains:

- What the project does.
- Why it exists.
- How to install it.
- How to use it.
- Technologies used.

A README is often the first file someone reads when visiting a project.

---

# Writing Documentation

Documentation is an essential part of professional software development.

Developers write documentation to:

- Explain how software works.
- Help others use their projects.
- Record important decisions.
- Support future maintenance.

Code explains **what software does**.

Documentation explains **why and how it works**.

Both are important.

---

# Tracking Changes

Software projects change constantly.

Developers use version control to manage these changes.

Git allows developers to:

- Track modifications.
- Create snapshots of progress.
- Review project history.
- Restore previous versions.

The basic workflow:

```text
Modify Files
    ↓
git status
    ↓
git add
    ↓
git commit
    ↓
git push
```

This creates a reliable history of how software evolves.

---

# Sharing Work

Modern developers rarely work in isolation.

GitHub allows developers to share and collaborate on projects.

Developers use GitHub to:

- Store repositories online.
- Share their work.
- Review changes.
- Collaborate with teams.
- Contribute to open-source projects.

A typical workflow:

```text
Local Repository
    ↓
Git Commit
    ↓
GitHub Repository
    ↓
Collaboration
```

---

# Maintaining Software

Software development does not end when an application is released.

Professional developers continue improving software through maintenance.

Maintenance includes:

- Fixing bugs.
- Improving performance.
- Adding features.
- Updating dependencies.
- Improving documentation.
- Updating security.

Software is continuously improved throughout its lifecycle.

---

# Professional Developer Workflow

A simplified professional workflow looks like this:

```text
Understand the problem
        ↓
Plan the solution
        ↓
Create project structure
        ↓
Initialize repository
        ↓
Develop features
        ↓
Track changes with Git
        ↓
Share work using GitHub
        ↓
Maintain and improve software
```

This process allows developers to build software that is organized, reliable, and easier to maintain.
