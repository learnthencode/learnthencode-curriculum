# Course Outline: Developer Foundations

## Course Overview

Developer Foundations is the first course in the LearnThenCode Full Stack Software Engineering Bootcamp.

This course introduces learners to the mindset, tools, and workflows used by professional software developers. Before building applications, learners first develop an understanding of how software is created, how developers work, and how modern development environments are structured.

The course focuses on building strong foundations in:

- Software engineering concepts.
- Development environments.
- Command-line usage.
- Version control.
- Git and GitHub workflows.
- Professional developer practices.

By the end of this course, learners should understand how professional developers organize their work and collaborate when building software.

---

# Course Philosophy

The purpose of Developer Foundations is not to teach learners every technology they will use throughout their careers.

Instead, this course focuses on developing the fundamental knowledge and habits required to become effective software engineers.

Learners should understand:

> Before building software, understand the environment, tools, and processes that make software development possible.

---

# Learning Outcomes

By the end of this course, learners should be able to:

- Explain what software engineering is.
- Differentiate between coding, programming, software development, and software engineering.
- Describe how computers execute software instructions.
- Understand the software development lifecycle.
- Set up a professional development environment.
- Use the command line to navigate and manage files.
- Explain the purpose of version control.
- Use Git to track changes in projects.
- Use GitHub to store and collaborate on software projects.
- Follow professional software development workflows.

# Lessons

## Lesson 1: Welcome to Software Engineering

### Purpose

Introduce learners to software engineering and help them understand the field they are entering.

### Topics

- What is software?
- What is software engineering?
- Coding vs programming.
- Software development vs software engineering.
- The software engineer mindset.
- The role of software engineers in solving problems.

---

## Lesson 2: Understanding Computers and Software

### Purpose

Build fundamental mental models about computers and how software works.

### Topics

- Computer hardware and software.
- Operating systems.
- Applications.
- Programming languages.
- How software instructions are executed.
- The relationship between humans and computers.

---

## Lesson 3: The Software Development Journey

### Purpose

Introduce the complete process of creating software products.

### Topics

- Identifying problems.
- Understanding requirements.
- Planning solutions.
- Software design.
- Development.
- Testing.
- Deployment.
- Maintenance.
- Iteration and improvement.

---

## Lesson 4: Development Environment Setup

### Purpose

Help learners prepare their computers for software development.

### Teaching Approach

Tools should be introduced when learners need them throughout the program.

This course focuses only on the essential tools required for starting software development.

### Topics

- Understanding development environments.
- Installing Visual Studio Code.
- Installing Git.
- Configuring Git.
- Creating a GitHub account.
- Verifying installations.

### Supported Operating Systems

Installation instructions should support:

- Windows.
- macOS.
- Linux.

---

## Lesson 5: Command Line Fundamentals

### Purpose

Introduce learners to the command-line interface used by software developers.

### Topics

- Terminal vs graphical user interface.
- Shells and terminals.
- Navigating directories.
- Creating files and folders.
- Moving and deleting files.
- Running commands.

### Practical Skills

Learners should be comfortable performing basic computer operations using the command line.

---

## Lesson 6: Files and Developer Organization

### Purpose

Develop professional habits for organizing software projects.

### Topics

- File and folder structures.
- Project organization.
- Naming conventions.
- Managing developer workspace.
- Keeping projects maintainable.

---

## Lesson 7: Introduction to Version Control

### Purpose

Help learners understand why developers use version control systems.

### Topics

- Problems with manual file versions.
- What is version control?
- Why Git was created.
- Repositories.
- Changes and history.
- Collaboration benefits.

---

## Lesson 8: Git Fundamentals

### Purpose

Teach learners the core Git workflow.

### Topics

- Creating repositories.
- Checking repository status.
- Tracking changes.
- Creating commits.
- Viewing history.
- Understanding branches.
- Managing changes.

### Practical Skills

Learners should be able to use Git confidently for personal projects.

---

## Lesson 9: GitHub and Collaboration

### Purpose

Introduce GitHub as a platform for storing and collaborating on software projects.

### Topics

- Remote repositories.
- Connecting Git and GitHub.
- Pushing code.
- Pulling updates.
- Cloning repositories.
- Issues.
- Pull requests.
- Collaboration workflows.

---

## Lesson 10: Professional Developer Workflow

### Purpose

Combine all previous skills into a professional development workflow.

### Topics

- Starting a software project.
- Creating project repositories.
- Writing documentation.
- Tracking changes.
- Sharing work.
- Maintaining projects.
